# Answer Bank Schema

Canonical schema for every entry stored in the Azure AI Search index and mirrored in the local `bank.jsonl` used by POC scripts.

## Field-by-Field

| Field | Type | Required | Indexed | Stored | Notes |
|---|---|---|---|---|---|
| entry_id | UUID v4 | yes | yes (key) | yes | Generated by `add_entry.py` on first insert; stable forever |
| question_text | string | yes | yes (searchable) | yes | Verbatim from Loopio or SME |
| canonical_question | string | yes | yes (searchable) | yes | Lowercased, punctuation-stripped, acronyms expanded |
| answer_text | string | yes | yes (searchable) | yes | Approved response body; no marketing adjectives |
| category | enum | yes | yes (filterable) | yes | See enum below |
| subcategory | string | no | yes (filterable) | yes | Free-form; lowercased |
| tags | string[] | no | yes (filterable) | yes | Free-form; lowercased |
| source | enum | yes | yes (filterable) | yes | `loopio_entry_id` \| `internal_sme` \| `correction` |
| source_loopio_entry_id | string | when source=loopio_entry_id | yes (filterable) | yes | Stable Loopio ID for round-trip sync |
| last_approved_date | ISO-8601 date | yes | yes (filterable) | yes | UTC; drives staleness flags |
| approved_by | email | conditional | no | yes | Required when source in {internal_sme, correction} |
| version | int | yes | yes (filterable) | yes | Monotonic per entry_id |
| deprecated_flag | bool | yes | yes (filterable) | yes | Default false; retired entries true |
| replaces | UUID[] | no | no | yes | Populated on successor when retiring a predecessor |
| certifications_referenced | string[] | no | yes (filterable) | yes | e.g. ["SOC2_TYPE2","ISO27001"] |
| pricing_reference_ids | string[] | no | yes (filterable) | yes | IDs into the pricing registry owned by `rfp-gates` |
| evidence_attachments | url[] | no | no | yes | SharePoint / Blob URLs |
| history | object[] | no | no | yes | Append-only prior versions (see below) |

### history[] object

| Field | Type | Notes |
|---|---|---|
| version | int | Prior version number |
| answer_text | string | Snapshot of the prior answer |
| last_approved_date | ISO date | Prior approval date |
| approved_by | email | Prior approver |
| retired_at | ISO datetime | When this version was superseded |
| change_note | string | Free-form reason for the supersede |

## Enums

### category

| Value | Scope |
|---|---|
| security | Infosec, auth, MFA/SSO, pen-test, vuln-mgmt |
| privacy | GDPR, CCPA, DPA, data residency |
| compliance | SOC 2, ISO 27001, HIPAA, PCI, FedRAMP |
| product | Feature support, roadmap (stable facts only) |
| integrations | APIs, webhooks, SSO/IdP, iPaaS |
| operations | Uptime, SLA, support, runbooks |
| commercial | Pricing model classes (not dollar amounts), licensing |
| legal | MSA, terms, liability caps (public-safe only) |
| company | Ownership, financials (public), headcount bands |
| other | Fallback; must be reviewed quarterly |

### source

| Value | Meaning |
|---|---|
| loopio_entry_id | Imported from the Loopio library via sync |
| internal_sme | Authored by an Contoso SME directly in the bank |
| correction | Landed via `merge_corrections.py` from `rfp-review` |

## Indexed vs Stored

- **Indexed** fields are part of the Azure AI Search query surface (searchable or filterable).
- **Stored-only** fields (`replaces`, `evidence_attachments`, `history`, `approved_by`) are returned with the document but not queryable. This keeps the index small and avoids leaking approver emails into search features.

## Relationships

| Relationship | Direction | Purpose |
|---|---|---|
| replaces[] | successor -> predecessor(s) | Audit chain when retiring; enables rollback |
| certifications_referenced[] | entry -> cert registry | `rfp-gates` verifies claim legitimacy |
| pricing_reference_ids[] | entry -> pricing registry | `rfp-gates` blocks un-approved pricing language |
| source_loopio_entry_id | entry -> Loopio row | Round-trip sync anchor |

## Versioning Rules

1. **Append-only.** Updating an entry preserves the prior version in `history[]`.
2. **Monotonic integer.** `version` increments by 1; no semver.
3. **deprecated_flag is the only retirement mechanism.** Hard delete is forbidden — we need the audit trail for SOC 2 and customer evidence requests.
4. **`replaces[]` is on the successor, not the predecessor.** This means predecessors never change when retired.
5. **No mutation of historical versions.** Corrections that target a past version create a new current version instead.
6. **`version=1` is reserved for first approval.** Draft entries (if any) live outside the bank until approved.

## Retention

- Deprecated entries remain in the index for at least 7 years to support audit.
- `history[]` retained for the lifetime of the entry_id.
- Deletions permitted only via written legal request with CISO sign-off.
